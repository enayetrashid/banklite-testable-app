# Functional Requirements

1. The system shall allow a demo user to sign in with a username and password.
2. The browser frontend shall call the login API instead of posting directly to service logic.
3. The dashboard shall load current balance data from the balance API.
4. The browser shall call deposit, withdraw, transfer, and transactions APIs using JavaScript.
5. The backend shall keep business rules in reusable service modules.
6. The backend shall store data in an in-memory storage class for demo purposes.
